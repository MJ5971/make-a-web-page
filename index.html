Topic 1: The Basics of the Web and HTML

How the Web Works

The World Wide Web or W.W.W for short was develop in the 1990s and has approximately 30 billion pages. Five components are used to interact with the Web. They are the User, Computer browser for example Chrome, Internet protocol HTTP, and finally servers which has host files.

HTML

Hyper Text Markup Language or HTML for short is used to create HTML Documents. HTML is written in a document but you have to follow HTML syntax and rules. These documents can be text, images, or videos and are used to create web pages. This site provides a good definition of an HTML document.

Elements, Attributes and Tags

An HTML Element is a component of the HTML document. Each element can have a specified HTML Attribute. Attributes such as class give elements meaning. These attributes are added to Tags HMTL Tags are the names of the Elements surrounded by brackets. Normally each opening tag has a closing tag. Tags without a closing tag are called a Void Tags

Why Computers are Stupid

Computers are stupid because they are not intuitive. The program must be detailed and precise to avoid inefficient use of time to debug it.

Inline vs Block Elements

Elements will be inline or block. In line elements are similar to a sentence in structure. Block elements encapsulates the element within a box.

Topic 2: Creating a Structured Document with HTML

Developer Tools (in the Browser)

The "tree-like structure" of HTML

The tree like structure represents the hierarchical relationship between elements. The elements themselves are contained in boxes. Each box can be a container for other boxes.

The relationship between indented HTML and boxes

Indented text in a HTML document depicts the position of the element's box. Greater indentations imply the position of the element box within other boxes

Topic 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Refraining from repetitive coding will reduce errors.

Repetitive editing will increase the likelihood of introducing erroneous code. This can occur when a definition or style is individually applied to the same element type for every instance that element appears.

The best way to refrain from repetitive editing is to use the same definition for common elements and a single source for their style.

CSS

Cascading Style sheets or (CSS) for short, resides in its own tab and applies style to an element. To refrain from repetitive coding this style can be applied to each instance of that element by referring its class.
